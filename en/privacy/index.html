OneByOne Privacy Policy

Last updated: May 29, 2026

OneByOne is an iPhone and Apple Watch app for running and recording personal routines, habits, and tasks one step at a time with AI assistance. The first release does not require a OneByOne account. Most app data is stored on device using Apple app storage technologies and may sync through the user's private iCloud/CloudKit storage when iCloud is enabled.

Data stored in the app

OneByOne stores routines, habits, tasks, routine run history, habit logs, metric logs, reflection logs, notification settings, calendar export settings, and a cached subscription entitlement state that users create or use in the app.

This data is stored using Apple app storage technologies. If iCloud is enabled for the user's Apple Account and the app can use iCloud, this data may sync through the user's private iCloud/CloudKit storage.

Remote AI processing

AI suggestions are a core OneByOne feature. By default, AI-powered features may call a OneByOne Supabase Edge Function to turn user input into an execution flow, and the server may send the request to a DeepSeek model.

When a user uses AI-powered features, OneByOne may transmit text the user enters, flow edit requests, selected candidate information, recent app context needed to generate a flow, app language, input language, timezone or time context, energy preferences, and, when location-based recommendations are used, the saved place type or saved place name. OneByOne does not intentionally include precise location coordinates in AI prompts.

For rate limiting, abuse prevention, error analysis, reliability, and quality checks, the Supabase backend may log a user ID if present in an authentication token, a rate-limit identifier derived from IP address or user-agent, rate-limit scope, function name, operation type, model, request status, app/input language, input and note character counts, input hash, execution ID, token counts, processing time, error information, and generated structured-result metadata.

Remote AI processing is used only to provide app functionality. It is not used for advertising or cross-app tracking, and OneByOne does not sell this data. Because AI-powered Begin generation and step adjustment are core features, remote AI processing is used by default when those features are used.

iCloud sync

OneByOne uses Apple's private CloudKit database for iCloud sync. Routine, habit, task, and log data may be stored in the user's private iCloud account. iCloud availability and storage are managed by the user's Apple Account and iOS settings.

OneByOne does not operate a separate developer-run sync server for this data.

Purchases

OneByOne Pro subscriptions and lifetime purchases are processed by Apple through the App Store. OneByOne uses StoreKit to check the current entitlement state and provide purchase and restore flows. Payment information is handled by Apple and is not collected by OneByOne.

Notifications

If the user enables reminders, OneByOne schedules local notifications on the device. Notification permission is managed in iOS Settings.

Calendar

If the user enables calendar export, OneByOne may request calendar access and create completed routine step events in a OneByOne calendar. OneByOne does not upload the user's full calendar data to Supabase, DeepSeek, or its own server.

Health data

If the user chooses a routine start condition based on health records, OneByOne may request access to health records so it can notify the user after sleep or workout updates. HealthKit data is processed on device, and raw HealthKit samples are not sent to Supabase or DeepSeek.

Location

If the user enables location-based recommendations, OneByOne may use location permission to match the current location with places saved by the user. When AI-powered features are used, a saved place name or broad place context may be included in the AI prompt. Precise coordinates are not intentionally sent to AI services.

Analytics, advertising, and tracking

OneByOne currently does not include third-party analytics SDKs, advertising SDKs, or cross-app tracking. The remote AI operational logs described above are used for app functionality, rate limiting, abuse prevention, error analysis, reliability, and quality checks.
